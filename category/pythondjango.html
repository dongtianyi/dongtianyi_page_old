use supervisor+nginx+virtualenv to deploy a django application

assume that you have virtualenv and a django application:

1 gunicorn 1.1 install in your virtualenv envirement: pip install gunicorn 1.2 add gunicorn to you django settings.py like: INSTALLED_APPS = [ ..., "gunicorn", ... ]

2 you can run your application by: python manage.py run_gunicorn here can send any parameter gunicorn accepted. please reference: https://docs.djangoproject.com/en/dev/howto/deployment/wsgi/gunicorn/

3 supervisor supervisor is daemon process. we use it to start/manage the command "python manage.py run_gunicorn". 3.1 install pip install supervisor 3.2 generate config files it’s highly recommended to simply generate config file of supervisor using command "echo_supervisord_conf". echo_supervisord_conf: It is in the directory you install supervisor.

    echo_supervisord_conf > /ect/supervisor.confg

    you can save you config file anywhere you can read it.

3.3 edit the file supervisor.confg your just generated.

    for simply, you just add the secion:

    [program:app_name]
    command=/path/to/your_command.sh
    user=user
    stdout_logfile=/path/to/supervisor_stdout.log
    redirect_stderr=true
    numprocs=1

    Note:
        command: the command you want to run, here we will use a sh file.
        user: to run the command's user name or id
        stdout_logfile: log file path
        redirect_stderr: err message to log into stdout_logfile
3.4 after you run "pip install supervisor", you can find in the install directory two command: supervisord and supervisorctl
    supervisord: start supervisor process
    supervisorctl: manage supervisor process

    run: 
        supervisord -c /ect/supervisor.confg
        the supervisor process will start.
    run:
        supervisorctl -c /ect/supervisor.confg
        you can enter the manage interface: use start, stop, reload etc. to manage the process.
